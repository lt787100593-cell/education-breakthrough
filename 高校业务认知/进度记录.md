## 二、当前严重问题（HTML结构损坏）

### 问题概述

index.html文件的HTML结构严重损坏，导致：
1. 教务管理页面tab切换失效
2. 多个页面出现错误的30秒概述卡片
3. 页面宽度问题反复出现

### 问题详情

#### 问题1：教务管理biz-detail容器被错误闭合

**位置**：index.html 第1244-1266行

```html
<!-- 第1244行：教务管理biz-detail容器开始 -->
<div class="biz-detail active" id="detail-teaching-affairs">
    <div class="biz-header">...</div>
    <div class="read-levels">...</div>
    
    <!-- 第1263行：30秒概述被简化为注释 -->
    <div class="read-content active" id="quick-teaching">
        <!-- 教务管理内容保持不变... -->
    </div>
</div>  <!-- 第1266行：biz-detail容器在这里被错误闭合！ -->
```

**影响**：
- 教务管理的5分钟了解（第2193行）和深入学习（第2397行）内容变成了孤立的元素
- 这些内容没有被包裹在biz-detail容器中
- JavaScript无法正确切换tab

#### 问题2：教务管理实际内容位置错误

**位置**：index.html 第2152行开始

教务管理的30秒概述、5分钟了解、深入学习的实际内容在第2152行开始，但它们是孤立的`<div class="read-card">`元素，没有被包裹在biz-detail容器中。

```html
<!-- 第2151行 -->
</div>
<!-- 第2152行：孤立的read-card开始 -->
<div class="read-card">
    <div class="read-label">30秒概述</div>
    <p class="read-lead">教务管理是高校教学运行的...</p>
    ...
</div>
```

#### 问题3：学生工作biz-detail容器重复

**位置**：index.html 第1269行和第3997行

学生工作的biz-detail容器出现了两次：
- 第1269行：有完整内容的容器
- 第3997行：只有占位符的容器

```html
<!-- 第1269行：学生工作biz-detail容器（有内容） -->
<div class="biz-detail" id="detail-student-affairs">
    ...
</div>

<!-- 第3997行：重复的学生工作biz-detail容器（只有占位符） -->
<div class="biz-detail" id="detail-student-affairs">
    <div class="placeholder">内容构建中</div>
</div>
```

#### 问题4：研究生培养biz-detail容器位置错误

**位置**：index.html 第3348行

研究生培养的biz-detail容器在第3348行，但它前面的内容（第2152-3347行）是教务管理的孤立内容。

```html
<!-- 第3347行 -->
</div>

<!-- 第3348行：研究生培养biz-detail容器开始 -->
<div class="biz-detail" id="detail-graduate-education">
    ...
</div>
```

### 问题根因分析

问题的根因是之前的编辑操作破坏了HTML结构：

1. 教务管理的biz-detail容器在第1266行被错误闭合
2. 教务管理的实际内容（第2152-3347行）变成了孤立的元素
3. 这些孤立的元素会显示在所有页面的顶部或底部
4. 学生工作的biz-detail容器被重复创建

### 修复方案

#### 方案1：删除第1266行的错误闭合标签

在第1266行删除`</div>`，让教务管理的biz-detail容器继续包含后续内容。

#### 方案2：重构HTML结构

重新组织HTML结构，确保：
1. 每个业务板块的biz-detail容器正确闭合
2. 所有read-content元素都在对应的biz-detail容器内
3. 删除重复的biz-detail容器

### 需要的操作

1. **备份当前index.html文件**
2. **修复教务管理biz-detail容器**：删除第1266行的错误`</div>`
3. **删除重复的学生工作biz-detail容器**：删除第3997-4005行
4. **验证修复效果**：检查所有业务板块的tab切换和宽度

```
[业务名称]是高校[定位描述]的核心业务，涵盖[核心职能1]、[核心职能2]、[核心职能3]等全流程。[主管部门]作为主管部门，承担着[核心使命]。

帆软在[业务名称]业务下的价值：[价值描述]。服务对象覆盖[服务对象]。

帆软核心优势：
1. 跨域协同：[针对该业务的定制化表达]
2. 主题数仓：构建[业务]主题数仓，[定制化表达]，对现有业务系统零入侵
3. 智能应用：从[自动化场景]到[智能化场景]，让数据主动服务[业务管理]
```

### 5分钟了解模板

```
一、业务概述（业务定义）
二、核心职能（3列表格：职能模块、核心工作、关键数据）
三、业务痛点（4个）
四、帆软价值场景（2x2卡片布局）
五、当前政策环境（3个政策）
六、关键岗位与决策链（5列表格：岗位、职责、关注点、切入话题、接触难度）
```

### 深入学习模板

```
一、业务全景架构
   - 业务定位（2段话）
   - 业务全景架构图（biz-ecosystem样式，底部整合帆软价值层）

二、核心职能详解（function-cards样式，6个模块）

三、业务流程详解（时间轴可视化）

四、业务发展历程（按实际情况分阶段，名称实事求是）

五、业务特色难点与帆软应对（★新增必选章节）
   - 该业务的核心难点/特殊性分析（结合实际业务场景，体现深度理解）
   - 帆软的能力边界（不硬编解决方案，坦诚面对行业共性问题）
   - 帆软的解决方案（咨询经验+产品能力，务实表述）
   - 典型案例场景（2个）

六、业务痛点根源分析

七、本科vs高职差异（对比表格）

八、关键岗位与决策链（5-6个岗位）

九、常见问题与应对（5-6个FAQ）
```

**模板说明**：
- 第五章"业务特色难点与帆软应对"是必选章节，每个专题都要有
- 内容要结合该业务的实际场景，体现对业务的深度理解
- 措辞要务实，不承诺无法解决的问题，坦诚面对行业共性难题
- 帆软的定位是"咨询+产品"，强调经验和方案设计能力，而非万能工具
